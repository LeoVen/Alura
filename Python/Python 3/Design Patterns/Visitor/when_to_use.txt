Podemos criar quantos visitors desejarmos, até mesmo um que exiba a expressão por extenso, por exemplo. Porém, para que o novo visitor funcione, ele precisa ter os devidos métodos de cada operação e para o número. Sabemos que pela natureza dinâmica da linguagem Python e do Duck Tying, o método aceita da expressões pode aceitar qualquer objeto que tenham esses métodos. Porém, nada nos impede de criarmos uma classe abstrata que contenha esses métodos como abstratos, obrigando a todos que herdarem a classe a implementarem esses métodos.
